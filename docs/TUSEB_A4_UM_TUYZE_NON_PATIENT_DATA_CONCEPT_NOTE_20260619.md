## One page concept

Working title: Türkiye Medical AI Safety Readiness Package

Problem: health AI teams can produce impressive model demos before they have a shared safety language for source support, Turkish clinical context, data fitness, human review, and public claim boundaries.

Aim: create a non patient data readiness package for Turkish medical AI safety that can support education, project preparation, and early route review without clinical deployment or clinical validation claims.

Scope:

1. Turkish medical LLM safety evaluation readiness.
2. Clinician AI literacy and responsible use micro module.
3. Health data quality and label audit worksheet.
4. Source support and no ranking failure mode intake.
5. Public claim hygiene for teams preparing reports, demos, or applications.

Work package 1: Source support and failure mode intake

Output: a short worksheet that asks whether a medical AI claim has source support, whether the output is appropriate for Turkish clinical context, and whether the example is being used as a safety signal rather than a model ranking.

Work package 2: Clinician review and literacy protocol

Output: a clinician facing review protocol that separates education, project preparation, and research readiness from patient care advice or clinical deployment.

Work package 3: Health data quality and label audit

Output: a data readiness worksheet using public or synthetic examples only. The worksheet checks label definition, leakage risk, missing context, population limits, annotation uncertainty, and documentation gaps.

Work package 4: Route ready public packet

Output: a compact public package with source basis, boundaries, validation script, and route questions that can be adapted if TÜSEB, TÜYZE, a university board, or a hospital education unit asks for more detail.
