## Decision states

### State 1. Recent acknowledgement window

Use this state when:

1. A route owner has acknowledged the material.
2. No substantive guidance has arrived.
3. The acknowledgement is still recent.
4. The next message would mainly repeat the same public link.

Allowed action:

1. Wait without new outreach.
2. Improve the public review packet.
3. Record the acknowledgement as waiting, not as endorsement.
4. Monitor Gmail before every new field action.

Blocked action:

1. Do not send a new message only to stay visible.
2. Do not claim that the route owner is reviewing in any public credibility statement.
3. Do not name the route owner in public as a supporter.

### State 2. One concise follow up after a real trigger

Use this state when one of these triggers appears:

1. The route owner asks for a shorter packet.
2. The route owner asks what decision is needed.
3. A public deadline makes route fit time sensitive.
4. A new artifact materially reduces the review burden.
5. A reasonable waiting period has passed and no duplicate message was sent.

Allowed action:

1. Send one concise follow up only after audit.
2. Ask for one of four outcomes: wrong route, better owner, not fit, or useful next review.
3. Link one public packet only.
4. Keep all boundaries visible.

Blocked action:

1. Do not attach a long package.
2. Do not imply urgency unless an official deadline creates it.
3. Do not ask for endorsement, approval, partnership, or validation.

### State 3. Stop and do not follow up

Use this state when:

1. The route owner says not fit.
2. The route owner gives no response after a bounded follow up.
3. The route becomes duplicative of another unanswered route.
4. The next message would require institutional claim, patient data, clinical deployment, payment, terms, or formal application authority.

Allowed action:

1. Close the route as no further action for now.
2. Keep building public infrastructure.
3. Reopen only if a new factual trigger appears.

Blocked action:

1. Do not send repeated outreach.
2. Do not infer rejection from silence.
3. Do not infer support from silence.

### State 4. Substantive reply received

Use this state when:

1. The route owner names a better owner.
2. The route owner asks a concrete question.
3. The route owner asks for a meeting.
4. The route owner asks for proposal, data, deployment, validation, ethics, budget, or institutional details.

Allowed action:

1. Stop new build work.
2. Summarize the reply.
3. Map the reply to the smallest truthful package.
4. Prepare response text only after source support and audit.

Blocked action:

1. Do not turn a question into endorsement.
2. Do not commit to partnership, patient data, deployment, validation, payment, terms, budget, or formal application without a separate decision gate.
