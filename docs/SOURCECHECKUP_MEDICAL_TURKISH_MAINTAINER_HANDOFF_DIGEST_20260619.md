## Maintainer tasks

### SCH001: intake triage

Action: decide whether a proposed Turkish medical AI safety sentence is a source support example, language support row, release gate row, outcome row, or release note closure row.

Allowed output: public review routing.

Blocked output: clinical approval, institutional approval, or model ranking.

### SCH002: source support handoff

Action: record whether a Turkish claim needs source support, language review, clinician review, data review, benchmark review, governance review, or release review.

Allowed output: review need named.

Blocked output: source proves safety, source truth certified, or evidence complete.

### SCH003: release gate handoff

Action: move rows from review state to blocked, unresolved, source support request, source checked, or wording risk removed.

Allowed output: decision state recorded.

Blocked output: reviewer approved, clinical validation complete, or deployment ready.

### SCH004: closure handoff

Action: name source, language, clinical, data, benchmark, institution, partner, release, and action boundaries before release notes are published.

Allowed output: closure state recorded.

Blocked output: all gates closed unless the source package proves each gate, which this public package does not do.

### SCH005: public action handoff

Action: publish only public GitHub artifacts that have validator, audit, and source support notes.

Allowed output: public commit, issue, release, or comment.

Blocked output: e mail send, formal application, TBYS action, PRODİS action, payment, terms acceptance, patient data use, ethics approval claim, clinical deployment, or clinical validation.

### SCH006: route owner reply handoff

Action: if a route owner replies, pause new public package work and handle the reply first.

Allowed output: route owner reply summary and decision packet.

Blocked output: treating silence as rejection or acknowledgement as endorsement.
